# Document Writing Guidelines

## Principles

- 문서는 한국어로 작성한다.
- 독자가 다음 작업을 바로 이어갈 수 있게 결론, 근거, 결정, 미확정 사항을 분리한다.
- 추측은 사실처럼 쓰지 않고 `확인 필요`, `가정`, `추정`으로 표시한다.
- 구현 코드와 문서는 섞지 않는다. 문서는 의사결정, 흐름, 제약, 검증 기준을 남긴다.
- 불필요한 배경 설명보다 재사용 가능한 판단과 근거를 우선한다.
- 비밀값, 토큰, credential, private URL, 개인정보는 기록하지 않는다.

## Concision

- 문서는 필요한 판단, 근거, 결정, 미확정 사항만 남긴다.
- 같은 내용을 제목, 요약, 본문, 체크리스트에서 반복하지 않는다.
- 독자가 이미 알 수 있는 일반론, 의례적인 배경 설명, 과한 도입 문구는 생략한다.
- 한 문단은 하나의 핵심만 다룬다.
- 목록은 실제 선택지, 조건, 절차, 체크 항목이 있을 때만 사용한다.
- `중요하다`, `고려해야 한다`처럼 정보량이 낮은 문장은 구체적인 영향이나 조치로 바꾼다.
- 문서의 각 섹션은 다음 중 하나에 기여해야 한다: 결정, 근거, 절차, 검증, 미확정 사항.
- 기여하지 않는 섹션이나 문장은 삭제한다.
- 초안 작성 후 중복 문장, 일반론, 다음 행동에 영향 없는 설명을 한 번 더 제거하고 저장한다.

## Recommended Structure

- 목적
- 배경
- 결정 사항
- 근거
- 대안
- 엣지케이스
- 검증 방법
- 미확정 사항
- 다음 단계

## Style

- 제목은 짧고 구체적으로 쓴다.
- 표는 비교, 필드 정의, 상태 전이에만 사용한다.
- Mermaid는 흐름이나 상태 전이가 복잡할 때만 사용한다.
- 오래 남길 문서는 날짜, 관련 파일, 관련 명령, 검증 결과를 포함한다.

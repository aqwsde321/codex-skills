# AGENTS.md Snippet

아래 내용을 전역 `~/.codex/AGENTS.md` 또는 프로젝트 `AGENTS.md`에 추가하면 짧은 한국어 호출어로 리뷰 skill을 사용할 수 있다.

```md
코드 리뷰 요청 시:
- 버그, 회귀, 엣지케이스, 테스트 정합성을 최우선으로 확인한다.
- 보안, 데이터 손상, 동시성, 성능 회귀처럼 실제 운영 리스크가 있으면 함께 지적한다.
- 스타일/취향 코멘트는 실제 유지보수 리스크가 있을 때만 언급한다.
- 발견 사항은 심각도순으로 파일/라인 근거와 함께 제시한다.
- 문제가 없으면 중요 이슈 없음과 남은 테스트 리스크만 간단히 말한다.

단축 리뷰 워크플로우:
- 사용자가 `커밋리뷰`, `리뷰커밋`, `막커밋 리뷰`, `마지막 커밋 리뷰하고 수정`, `$review-fix-commit`처럼 요청하면 `$review-fix-commit` skill을 사용해 "새 에이전트로 마지막 커밋을 리뷰하고, 문제 있으면 수정하고, 관련 테스트를 실행한 뒤 한국어 Conventional Commit으로 커밋한다"는 명시 요청으로 해석한다.
- 사용자가 `변경리뷰`, `작업트리 리뷰`, `워킹트리 리뷰`, `현재 변경 리뷰`, `$review-fix-changes`처럼 요청하면 `$review-fix-changes` skill을 사용해 "새 에이전트로 현재 git 변경분(staged/unstaged/untracked)을 리뷰하고, 문제 있으면 수정하고, 관련 테스트를 실행한다"는 명시 요청으로 해석한다.
- 사용자가 `반복변경리뷰`, `반복리뷰`, `끝까지 리뷰`, `수정할 거 없을 때까지 리뷰`, `$review-fix-until-clean`처럼 요청하면 `$review-fix-until-clean` skill을 사용해 "`$review-fix-changes` 한-pass 리뷰/수정/테스트를 fresh review pass에서 수정할 actionable finding이 없다고 할 때까지 반복한다"는 명시 요청으로 해석한다.
- 사용자가 `반복커밋리뷰`처럼 요청하면 `$review-fix-until-clean` skill을 사용하되 마지막 커밋 기준 반복 리뷰/수정/테스트/커밋 워크플로우로 해석한다.
- 새 에이전트는 리뷰와 제한된 개선 작업을 맡기고, 최종 diff 확인/테스트는 메인 에이전트가 수행한다. 커밋이 필요한 워크플로우에서는 메인 에이전트가 커밋한다.
- 현재 변경분 리뷰는 기본적으로 커밋하지 않고, 사용자가 `커밋까지` 또는 `커밋해줘`라고 명시한 경우에만 커밋한다.
- 반복리뷰는 기본적으로 현재 변경분 기준이며, 현재 변경분 회차는 `$review-fix-changes`를 재사용한다. 기본 최대 반복은 10회이고, 사용자가 `최대 5번`, `5회까지만`, `max=5`, `iterations=5`처럼 명시하면 그 값을 사용한다. 사용자가 `마지막 커밋` 또는 `커밋 기준`이라고 명시하면 마지막 커밋 기준으로 시작한다. 같은 finding이 반복되거나 최대 반복에 도달하면 무한 반복하지 말고 남은 리스크를 보고한다.
- `리뷰만`이라고 하면 수정/커밋하지 않는다.
```
